Модель корпуса для печати на 3D-принтере

  При возможности приобретения недорогих корпусов печатать их на 3D-принтере смысла нет, тем более, что мелкие детали корпуса требуют использования принтера с высоким разрешением, а такая печать требует времени и обходится недёшево.
  Для печати на 3D-принтере можно скачать файл STL 3D-модели корпуса.
  Части корпуса могут скрепляться четырьмя полосками стеклотекстолита толщиной 0,5мм - мне показалось это более надёжным при многократной сборке-разборке корпуса, чем пластиковые стойки.
  Полоски плотно вставляются в пазы в вертикальных стенках.
  Чтобы можно было использовать полоски, разрешение печати должно быть высоким или толщину полосок понадобится подбирать после печати с учётом неточности 3D-печати.
  Высококачественная и дорогостоящая 3D-печать с разрешением 0,025мм обеспечит ширину паза после печати 0,500±0,050мм.
  Проще и дешевле использовать клей, который позволит при необходимости рассоединить части корпуса. 
  Здесь можно увидеть корпус адаптера в 3D-анимации
В этот корпус можно ввести кабель диаметром 3,5мм. При необходимости пользователь может увеличить вырез в стенке для установки амортизирующего кабельного ввода. Для этого в стенке предусмотрены выемки под размеры покупного кабельного ввода.

"Корпуса" из термоусадки

 Для личного применения вполне достаточно корпуса из обычной термоусадочной трубки с усадкой  2:1 и начальным диаметром 12мм.

 В адаптере на фото ввод кабеля выполнен из нескольких термоусадочных трубок. 
 Первая трубка самая короткая, после её усадки надевается трубка подлиннее.

Трубки выравниваются по краю платы адаптера. Каждая следующая трубка длиннее предыдущей, что обеспечивает
  • сглаживание переходов за счёт того, что каждая следующая, более длинная трубка накрывает срезы всех предыдущих трубок,
  • постепенное уменьшение жёсткости кабельного ввода по мере удаления от адаптера.
Такой кабельный ввод лучше покупного, т.к. покупной может быть либо недостаточно жёстким по сравнению с кабелем, либо существенно жёстче кабеля, тогда кабель будет изгибаться/ломаться при выходе из кабельного ввода.

  Если нужен более крепкий корпус, подойдет клеевая термоусадочная трубка с усадкой 3:1 или 4:1 и начальным диаметром 12мм.
 Плату сначала помещают в обычную термоусадку 2:1 - достаточно поместить только плату, разъем USB не накрывать, а со стороны  
кабеля обрезать термоусадку по границе платы. Эта термоусадка защитит компоненты на плате от клея термоусадки 3:1, воздействие которого  неизвестно.
 Клеевая термоусадка 3:1 должна частично накрывать разъем USB и обрезаться со стороны ввода кабеля с запасом в несколько миллиметров.
 После остывания термоусадка 3:1 становится довольно прочной и толстой (порядка 1мм).

Программатор UPDI и адаптер на ATmega808

(микроконтроллер megaAVR 0-series)

Для микроконтроллеров megaAVR 0-series трудно найти дешёвый программатор с удобным в работе управляющим ПО - предлагаются дорогие универсальные программаторы как для микроконтроллеров AVR, так и для PIC, поддерживающие набор интерфейсов программирования всех серий этих микроконтроллеров.
На GitHab можно разыскать исходники приложений для работы с программатором. Эти программы запускаются из командной строки и требуют ввода ряда параметров..
Можно найти и рекомендации по изготовлению программатора из адаптера USB<>UART с небольшой доработкой.

К сожалению, в сети пока мало примеров программ для микроконтроллеров megaAVR 0-series, а организация и наименование регистров, а также управление узлами микроконтроллера значительно изменились и требуют изучения.

 На фото самодельный программатор UPDI, вернее, это адаптер USB UART с дополнительными элементами, позволяющими организовать программатор UPDI.
 Он может работать и как адаптер USB UART, например, для программирования WiFi модулей ESP8266 и ESP32, а также микроконтроллеров непосредственно из IDE Arduino.
 
На  фото программатор UPDI подключен к USB компьютера
 и к программируемому микроконтроллеру ATmega808 на плате адаптера.

 Напряжение питания +3,3В на адаптер поступает с программатора.

 Можно посмотреть полное видео программирования по UPDI на youtube

 
 

















На компьютере выполняется приложение, загружающее программу в микроконтроллер,
В данном случае это программа-Bootloader размером 256 байт для микроконтроллера ATmega808.

  gif-картинка демонстрирует
  -считывание состояния фьюз-байта 8,
  -запись в него "1" ( 1 * 256 байт - один boot-сектор )
  -выбор hex-файла с прошивкой (Bootloader.hex)
  -запись 4-х страниц по 64 байта (всего 256 байт; 64 байта - размер страницы памяти ATmega808)
  -автоматическое отключение устройства USB после загрузки
  -подключение устройства к USB по нажатии кнопки ENABLE
  -считывание 4-х страниц по 64 байта для контроля прошивки в микроконтроллере.

  Рабочие программы будут загружаться ЭТИМ ЖЕ ПРИЛОЖЕНИЕМ уже через Bootloader, загруженный в микроконтроллер,
  при этом программатор UPDI уже не требуется - он отключается от адаптера и от USB,
  а сам адаптер подключается к USB компьютера,

Применение I2C SPI 1-Wire USB адаптеров

Адаптер I2C SPI 1-Wire USB и MEMS-акселерометры с интерфейсом SPI.

 Плата малошумящего MEMS-акселерометра с интерфейсом SPI подключена к адаптеру I2C SPI 1-Wire USB на микроконтроллере ATmega8A пятипроводным кабелем длиной 7 метров.

 Плата акселерометра находится на линейке, между метками 8 и 9см. Габариты платы акселерометра 4х13мм, есть отверстие для крепежа.
 Плата односторонняя и может быть закреплена на исследуемом объекте клеем - в этом случае фрагмент платы с крепежным отверстием можно отрезать, уменьшив длину платы до 9мм (габариты 4х9мм).
 Кабель выполнен из провода МГТФ 0,03, помещённого в мягкую термоусадочную трубку.
 Преимущества по сравнению с акселерометрами RS-485 : минимальный вес платы акселерометра и гибкость кабеля обеспечивают минимальное механическое воздействие на объект исследования, к которому крепится акселерометр; недостаток - мЕньшая помехоустойчивость: при сильных электрических помехах (станки, сварка) интерфейс RS-485 будет работать надежнее.

  Для работы микросхемы MEMS-акселерометра c адаптером I2C SPI 1-Wire USB достаточно использовать соответствующее выбранному акселерометру компьютерное приложение,
 а микросхему MEMS-акселерометра с одним-двумя конденсаторами разместить на отдельной плате.
 

Применение адаптера позволяет оперативно заменять акселерометры, например, для измерения слабых и сильных ускорений подключать платы с разными акселерометрами.

Вес и габариты отдельной платы с датчиком (акселерометром или гироскопом) гораздо ниже веса и размеров платы, на которой, кроме датчика, расположены и контроллер с разъёмом.

С адаптером можно использовать отдельную маленькую и лёгкую плату с датчиком. Кабель можно сделать максимально лёгким и гибким.

Лёгкая малогабаритная плата, лёгкий и гибкий кабель дают возможность закрепить плату с датчиком на минимальной площади и окажут минимальное механическое воздействие на исследуемый объект. Это особенно важно при измерениях на объектах с небольшой массой или, например, на свободно качающихся объектах - чем меньше масса акселерометра и мягче и легче кабель, тем меньше вносимое ими затухание.


   К адаптеру I2C SPI 1-Wire USB могут подключаться MEMS-акселерометры с интерфейсом SPI, в том числе те, которые установлены в :акселерометрах USB   или в  прецизионном акселерометре;
  например, популярный акселерометр ADXL345 c диапазоном измеряемых ускорений ±16g  (±2g / ±4g / ±8g / ±16g), совместимый с ним акселерометр ADXL375 с диапазоном ±200g (±25g / ±50g / ±100g / ±200g) .
  Каждый тип MEMS-акселерометра требует отдельного программного обеспечения - как для микроконтроллера, так и для компьютера.
 
Для увеличения расстояния от компьютера до адаптера, а значит, и до датчиков еще на 1,8 метра можно дополнительно использовать стандартный кабель-переходник USB-A-вилка < > USB-A-гнездо.

 Адаптер I2C SPI 1-Wire USB с I2C-датчиками влажности, давления и температуры высокой точности.

 
 В белой оболочке - обычной термоусадке -

 адаптер I2C SPI 1-Wire USB

 К его контактам припаян 4-проводный кабель I2C длиной 3 метра с датчиком температуры,

 к датчику температуры подсоединён второй кабель длиной 1 метр с подключенными к нему через разъём и группу контактов датчиками давления и влажности.

 Можно дополнительно использовать стандартный кабель-переходник USB-A-вилка < > USB-A-гнездо для увеличения расстояния от компьютера до датчиков еще на 1,8 метра.

Модуль GPS с интерфейсом UART и Адаптер I2C SPI 1-Wire USB на микроконтроллере megaAVR 0-series.

 
 К адаптеру I2C SPI 1-Wire USB на базе микроконтроллера megaAVR 0-series подключен модуль GPS.
 Представленный на фото микроконтроллер megaAVR 0-series в 28-выводном корпусе поддерживает три аппаратных канала USART.
 Микроконтроллер получает текстовую информацию, передаваемую модулем GPS, выделяет из неё необходимую часть и преобразует в цифровые данные (координаты, время и т.д.) для дальнейшего использования,
 например, для управления рулём карпового кораблика.
 
 Один канал USART микроконтроллера используется для связи с компьютером по USB.
 
 Ещё один канал USART выведен на контакты микроконтроллера, но если для работы устройства необходимо использовать низкочастотный кварцевый резонатор 32кГц  или 32768Гц, эти же контакты будут использоваться для подключения кварцевого резонатора.
 Снизу платы зарезервированы контактные площадки для подключения либо кварцевого резонатора, либо внешнего устройства с интерфейсом UART.
 
 Третий канал USART использует те же линии одного из каналов интерфейса SPI, который использовался для подключения акселерометра (см.предыдущее фото).
 К этому каналу подключен модуль GPS.
 
 Выбор узла микроконтроллера (здесь это USART или SPI), к которому подключены контакты, выполняется программно - микроконтроллер поддерживает мультиплексирование контактов. 

Модули 5- и 6-разрядного семисегментного индикатора и Адаптер I2C SPI 1-Wire USB на микроконтроллере megaAVR 0-series.

Это лишь несколько примеров применения адаптеров I2C SPI 1-Wire USB. Их можно использовать для подключения самых разных устройств с интерфейсами I2C и SPI к компьютеру - микросхемы памяти, часы реального времени, различные датчики, свои устройства и т.д.
Например, на базе адаптера можно организовать метеостанцию с долговременной автономной записью на SD карту (интерфейс SPI) температуры, влажности, давления (датчики с интерфейсами I2C) и привязкой к реальному времени (высокоточные термостабильные часы реального времени с интерфейсом I2C).
