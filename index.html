I2C SPI 1-Wire USB адаптер

Адаптеры I2C SPI 1-Wire < > USB на базе микроконтроллеров STM8S003 и ATmega8A/ATmega328P, а также ATmega808 (новая megaAVR 0-series), обеспечивают подключение к компьютеру различных устройств или микросхем с интерфейсами I2C, SPI и 1-Wire.
а для ATmega808 и с интерфейсом UART.
Адаптеры можно использовать и как устройства под ключ для собственных систем, и как наборы разработчика (Starter Kits) для освоения микроконтроллеров - как для программистов, желающих быстро перейти на новую серию микроконтроллеров, так и для тех, кому достаточно среды разработки Arduino.
Особенно актуально освоение микроконтроллеров megaAVR 0-series (ATmega808/ATmega809  и др.):
  • у них гораздо шире возможности, чем у ATmega8/ATmega328P,
  • выпускаются ATmega808 не только в корпусе TQFP32, но и в TQFP48, и в SSOP28 (используется в адаптере),
  • продаются они существенно дешевле.
При этом их программирование существенно отличается от ATmega8, а готовых дешевых программаторов, совместимых со средой разработки Microchip Studio, не выпускают.

Я регулярно использую такой адаптер для тестирования самых разных устройств I2C, SPI, 1-Wire или UART и отладки программного обеспечения.
Проверенный адаптер и ПО, отлаженное в плане передачи/приёма данных по интерфейсам I2C, SPI и 1-Wire, значительно облегчают освоение новых устройств.
Остается разобраться по документации нового подключаемого устройства, какие команды и по какому адресу отправлять и из каких регистров считывать результаты.

Для упрощения подключения плат с различными устройствами к плате адаптера, используемого для тестов, я припаял короткий кабель с разъёмными контактами







 Адаптер I2C SPI 1-Wire<>USB - законченное компактное устройство в корпусе,

которое пользователь может применить в составе собственных систем,
не тратя времени на изготовление платы, подбор корпуса и разработку ПО, 
что обеспечивается
- совместимостью плат адаптеров со стандартными недорогими корпусами;
- возможностью одновременного подключения к адаптеру нескольких устройств с разными интерфейсами - I2C, SPI и 1-Wire;
- разнообразием программного обеспечения в библиотеке примеров;
- простотой программирования адаптеров на базе микроконтроллеров ATmega8/ATmega328P и ATmega808 - при уже прошитом Bootloader-е загрузка прошивок для этих микроконтроллеров выполняется без программатора, по USB;
- возможностью перехода в режим программирования и в режим работы по команде компьютера; при возврате к режиму работы компьютерное приложение Bootloader можно не закрывать, оно может отключаться от адаптера автоматически, а для возврата к загрузке через Bootloader надо лишь нажать кнопку в этом приложении;
- совместимостью со средой разработки Arduino (удобно для начинающих: много примеров).

То есть,
  для перехода в режим программирования не обязательно замыкать контакты на плате адаптера перед подключением к USB, а для возврата к выполнению рабочей программы не требуется отключение-подключение адаптера.
  Можно переключаться между приложениями, не перезапуская приложение Bootloader перед каждым программированием.
При многократных загрузках прошивок во время доработки программы это серьёзные удобства.

Для производства собственной системы на базе адаптера разработчик может воспользоваться ПО из библиотеки примеров или написать для адаптера собственное ПО.
Использование адаптера в своих разработках ощутимо сокращает сроки работ и затраты: не надо разрабатывать и заказывать плату, монтаж (стоимость такого заказа превысит стоимость адаптера, не говоря о потраченном времени), искать или разрабатывать корпус, а универсальное программное обеспечение адаптера подойдет для решения многих задач.

Для облегчения применения I2C SPI 1-Wire USB адаптера
  • пополняется библиотека примеров программного обеспечения для работы с различными датчиками и устройствами (акселерометры, датчики температуры, влажности, давления, часы реального времени, дисплеи, память, модуль GPS, ИК-термометр и т.д.);
  • предусмотрено размещение адаптеров на базе любого из перечисленных выше микроконтроллеров в корпусе для малогабаритных устройств USB, продаваемом на aliexpress:
  • проверен простейший вариант достаточно прочного корпуса: адаптер помещается в клеевую термоусадочную трубку начальным диаметром 12мм с коэффициентом усадки 3:1;
  • подготовлен файл stl для 3d печати - корпус для адаптера можно распечатать на 3d принтере;     ВИДЕО корпуса USB 3d
Для быстрого освоения адаптера I2C SPI 1-Wire USB предоставляется следующее программное обеспечение:
  • универсальная программа для микроконтроллеров (прошивка и исходники) – она позволяет работать с интерфейсами под управлением команд, поступающих в адаптер по USB; 
  • компьютерное приложение под Windows для загрузки прошивок через bootloader;
  • компьютерные приложения для управления адаптером (готовые приложения и исходники - проекты в Delphi для работы с драйверами прямого доступа и виртуального COM-порта).

От адаптеров, которые представлены в сети Internet, адаптер I2C SPI 1-Wire USB отличается тем, что
  • работает в режиме ВЕДУЩЕГО (MASTER) - часть предлагаемых в Internet адаптеров не может работать в этом режиме;
  • напрямую подключается к разъему USB компьютера (разъём USB-A);
  • плата адаптера разработана под продающиеся недорогие корпуса и может поставляться разработчиками в корпусе, как готовое устройство; 
  • предлагаются готовые примеры программного обеспечения для конкретных устройств;
  • есть драйверы для разных операционных систем.
  • к адаптеру можно одновременно подключить 
    • несколько устройств с интерфейсом I2С и до трёх устройств с интерфейсом SPI 
      • или
    • несколько устройств с интерфейсом I2С, до двух устройств с интерфейсом SPI и несколько устройств с интерфейсом 1-Wire,
      • или
    • до четырёх устройств с интерфейсом SPI и несколько устройств с интерфейсом 1-Wire,
      • или
    • до пяти устройств с интерфейсом SPI.
7-разрядное адресное пространство интерфейса I2C позволяет оперировать 128-ю подключенными устройствами;
64-разрядное адресное пространство интерфейса 1-Wire теоретически позволяет работать с 2^64 (два в 64-й степени) устройствами.
В обоих случаях количество устройств будет ограничиваться скорее длиной кабеля и током, потребляемым этими устройствами.

Незадействованные линии можно использовать в качестве входов/выходов общего назначения (general I/O).
Для подключения кабеля в адаптере предусмотрено 12 контактов.
8 из них подключены к сигнальным линиям:
3 линии - для интерфейса SPI,
1 - для выбора устройства при работе с SPI (каждое устройство, подключаемое к интерфейсу SPI адаптера, требует отдельной линии выбора этого устройства);
2 - для интерфейса I2C, если I2C не использовать, можно подключить дополнительно 2 устройства к интерфейсу SPI;
1- вход/выход общего назначения (I/O), можно использовать для подключения одного устройства SPI;
1 - для интерфейса 1-Wire, если 1-Wire не использовать, можно подключить ещё одно  устройство SPI.

Для чего нужна возможность подключения нескольких устройств к разным интерфейсам одновременно?
Пример системы:
к адаптеру подключаются датчики влажности, давления и часы реального времени по I2C интерфейсу, несколько датчиков температуры по интерфейсу 1-Wire, SD карта по интерфейсу SPI. При соответствующей программе такое устройство может длительно записывать параметры окружающей среды на SD карту с привязкой ко времени без команд компьютера. Если линию прерывания часов реального времени подключить ко входу/выходу общего назначения, можно удерживать микроконтроллер в спящем режиме и пробуждать его по прерыванию от этих часов, а после считывания показаний датчиков и записи их на SD карту снова переводить микроконтроллер в режим сна. Перевод микроконтроллера в режим сна значительно сокращает средний потребляемый ток и продолжительность работы автономного источника питания.
Нет особого смысла заниматься снижением потребления тока при питании по USB от компьютера, но в предлагаемой конфигурации можно полностью отладить программу микроконтроллера для автономной работы с энергосбережением.
Окончательный вариант автономного устройства требует размещения в едином корпусе с адаптером батареи питания и SD карты.

Адаптер I2C SPI 1-Wire USB снабжён разъемом USB-A для подключения к компьютеру или ноутбуку.
Для подключения адаптера I2C SPI 1-Wire USB к планшету понадобится переходник.

Программное обеспечение адаптера I2C SPI 1-Wire USB включает примеры работы с датчиками температуры, влажности, давления, часами реального времени, ЖК-дисплеем с интерфейсом I2C, с модулем GPS,
с инфракрасным термометром, дисплеем и аккумулятором в автономном режиме,
а также с акселерометрами и памятью с интерфейсом SPI
и с популярным датчиком температуры DS18B20 с интерфейсом 1-Wire.
Список примеров работы с устройствами постоянно расширяется.

Пользователи могут загружать собственную прошивку в микроконтроллеры адаптеров I2C SPI 1-Wire USB:

  • адаптер I2C SPI 1-Wirе USB на базе микроконтроллера STM8S003

                перепрограммируется пользователем по однопроводному интерфейсу (SWIM). Для программирования микроконтроллера STM8S003 подойдет недорогой программатор, например, ST-link-mini V2;

  • адаптер I2C SPI 1-Wire USB на базе микроконтроллеров ATmega8A, ATmega808

                перепрограммируется пользователем по USB через прошитый в микроконтроллер загрузчик (bootloader) - программатор для этого не требуется.
               
При необходимости загрузить собственную прошивку в микроконтроллер ATmega8A/ATmega328P адаптера I2C SPI 1-Wire USB можно и по интерфейсу SPI, воспользовавшись дешевым программатором, например, каким-либо AVR ISP для программирования микроконтроллеров AVR. Однако для этого понадобится предварительно отключить внешние устройства, подключенные к интерфейсу SPI адаптера I2C SPI 1-Wire USB (отпаять кабель от 6-ти контактов адаптера I2C SPI 1-Wire USB).  
При полном стирании (Erase)  памяти программ микроконтроллера, которое обычно выполняется перед программированием, будет стёрт и bootloader.
Дальнейшее программирование ATmega8A будет возможно только по интерфейсу SPI с использованием программатора, так как прошивка bootloader отдельно не предоставляется. Это же касается и микроконтроллера ATmega808, но он требует программатора с интерфейсом UPDI.
